## 一、最重要的一条:先看真画面,再改代码

今天我最大的错误是 **在看不见游戏画面的情况下凭猜测改代码** → 你跑 → 还是坏 → 再猜。
这个"盲改循环"浪费了很多来回。

✅ 正确做法(今天反复验证有效,几乎一击命中):
1. **非侵入抓当前帧**:`_capture_client_via_printwindow(hwnd)`(只截图,不动鼠标)。
2. **用真实 OCR 跑一遍**:`classify_hybrid` → `parse_X` → `policy.decide`,
   看 bot **到底把画面识别成什么、解析出什么、最后决定点哪**。
3. **看清根因再动手改**。

> 角色界面卡死、获得奖励、布谷鸟时钟、训练按钮点偏、训练检视——全是这么定位掉的。

## 三、改完代码必须重启 bot(Python 不热加载!)

今天很多"修了还是坏"极可能是因为 **运行进程还是旧代码**。
任何代码改动后,**停掉进程重新跑** 才生效。否则测的是老逻辑,白忙。

## 四、沟通要点

**1. 术语对齐**(今天我因此误判过一次:以为训练类型对不上代码,其实对得上):

| 你的说法 | 游戏显示 | 代码内部 |
|---|---|---|
| 力量 | 力量 | power |
| 生命 | 体力 | stamina |
| 防御 | 韧性 | guts |
| 命中 | 专注 | wisdom |
| 命抗 | 保护 | speed |

**2. "慢"要分清是哪一种**:
- 识别慢 = OCR 整屏分类(~0.7–1.1s/帧,硬成本)
- 点击慢 = 连点频率 / 每帧点几下
- 这是两个不同的旋钮,别调错(今天我把识别慢当点击慢去调,过度加速到 20Hz)。

## 九、有用的技术备忘(本游戏特有)

- **"旅程起点" 标题被 3 个界面共用**(角色选择 / 祝福配置 / 旅程起点)→ 纯 OCR 分不开,
  必须用 `classify_hybrid`(视觉辅助)区分;live loop 默认已走 hybrid。

- **预览加成(+N)只在训练被选中时显示在左侧面板** → 要逐个点开看(像小祝福)。
