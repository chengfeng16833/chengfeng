# 笔记本上手说明

这套脚本不写死任何路径，把整个仓库文件夹放到笔记本任意位置都能用。

## 前置条件

1. 安装 **Python 3.11+**：https://www.python.org/downloads/
   - 安装时务必勾选 **「Add Python to PATH」**
2. 把本仓库下载/克隆到笔记本（任意目录均可）：
   ```
   git clone https://github.com/chengfeng16833/chengfeng.git
   ```

## 三步走

| 步骤 | 双击运行 | 作用 |
|------|----------|------|
| 1. 准备环境 | `笔记本-准备环境.bat` | 装依赖 + 跑测试，确认环境就绪 |
| 2. 跑测试   | `笔记本-跑测试.bat`   | 随时复跑单元测试 |
| 3. 启动 GUI | `笔记本-启动控制台.bat` | 打开全局控制台 |

第一次只需双击 **`笔记本-准备环境.bat`**，看到「全部测试通过，环境就绪!」即可。

## 脚本做了什么（自动处理）

- 自动定位仓库目录（脚本所在目录），不用手改路径
- 自动探测 Python（依次尝试 `python` / `py -3` / `python3`）
- 装依赖优先用清华镜像，失败自动回退官方 PyPI
- 跑测试只扫 `tests/` 目录，避开 `tools/` 里会触发真实 OCR 的调试脚本

## 和台式机脚本的区别

根目录原有的 `启动控制台.bat` / `跑测试.bat` 写死了台式机路径
（`C:\Users\ChengFeng\...` 和 Codex 自带 Python），笔记本上用不了。
请用上面带「笔记本-」前缀的三个脚本。

## 出问题怎么办

- 提示「没找到 Python」→ 重装 Python 并勾选 Add to PATH
- 依赖装不上 → 检查网络；脚本已自动尝试镜像+官方源两条路
- 测试有红 → 把 `.bat` 窗口里的报错截图发我
