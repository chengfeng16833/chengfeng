# 环境上手说明 (台式机/笔记本通用)

根目录的三个 `.bat` 已全部改成**可移植版**：不写死任何路径，把整个仓库
文件夹放到任意位置都能用，台式机和笔记本共用一套。

## 前置条件

1. 安装 **Python 3.11 / 3.12 / 3.13**（⚠️ **不要装 3.14**）：https://www.python.org/downloads/
   - 原因：依赖 `paddlepaddle 3.2.2` 只提供到 **Python 3.13** 的安装包，3.14 装不上。
   - 安装时务必勾选 **「Add Python to PATH」**
   - 脚本会自动优先挑 3.13/3.12/3.11，所以即使你电脑上同时装了 3.14 也没关系。
   - (台式机若已有 Codex 自带运行时，脚本会自动优先使用，无需额外装)
2. 把本仓库下载/克隆到电脑（任意目录均可）：
   ```
   git clone https://github.com/chengfeng16833/chengfeng.git
   ```

## 三个脚本

| 双击运行 | 作用 |
|----------|------|
| `准备环境.bat`   | 一键：装依赖 + 跑测试，确认环境就绪（**第一次只需这一步**） |
| `跑测试.bat`     | 随时复跑单元测试 |
| `启动控制台.bat` | 打开全局控制台 GUI |

第一次双击 **`准备环境.bat`**，看到「全部测试通过，环境就绪!」即可。

## 脚本自动处理了什么

- 自动定位仓库目录（脚本所在目录），不用手改路径
- 自动探测 Python：先用 Codex 自带运行时（台式机），找不到再依次回退
  `python` / `py -3` / `python3`
- 装依赖优先清华镜像，失败自动回退官方 PyPI
- 跑测试只扫 `tests/` 目录，避开 `tools/` 里会触发真实 OCR 的调试脚本

## 出问题怎么办

- 提示「没找到 Python」→ 重装 Python 并勾选 Add to PATH
- 依赖装不上 → 检查网络；脚本已自动尝试镜像+官方源两条路
- 测试有红 → 把 `.bat` 窗口里的报错截图发我
