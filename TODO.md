# Starsavior Trainer — 需求文档

> 给接手此项目的 AI：
> 这是一份 Starsavior PC 游戏训练自动化脚本的需求。请根据截图识别每个画面的 UI 布局，
> 补充 `config/regions/2560x1440.json` 中的区域坐标，并在 `screen_reader.py` 中实现
> 对应的画面读取逻辑。决策规则（policy）已在 `policy.py` 中基本完成，无需大改。

---

## 1. 项目概述

**目标**：自动操作 Starsavior PC 客户端，完成一次完整的训练 run。
**流程**：截图 → 识别当前画面 → 提取 OCR/颜色信息 → 决策 → 点击
**分辨率**：2560×1440（游戏客户端区域，不含 Windows 标题栏）
**技术栈**：Python + PaddleOCR + PIL/OpenCV + PyAutoGUI

### 关键代码文件

| 文件 | 职责 | 是否需要修改 |
|------|------|-------------|
| `starsavior_trainer/models.py` | 所有数据模型（Screen 枚举、Observation、Action、各画面 Payload） | 可能需要加新类型 |
| `starsavior_trainer/policy.py` | 决策逻辑：根据 Observation 产出 Action | **基本不需要改** |
| `starsavior_trainer/screen_reader.py` | 从截图中 OCR 读取各画面的结构化信息 | **需要大量补充** |
| `starsavior_trainer/vision.py` | 颜色检测（蓝色按钮、彩色光环等） | 可能需要补充 |
| `config/regions/2560x1440.json` | 每个画面的按钮/文字区域的像素坐标 [x, y, w, h] | **需要大量补充** |

### 工作原理

```
截图 → classify_screen(OCR锚点) → Screen类型 → ScreenReader读对应区域 → Payload → Policy决策 → Action(click坐标)
```

### 3.9 训练五选一 (TRAINING_SELECT) ❗ 需补区域坐标

**触发**：从 Training Hub 点击 `训练`
**截图描述**：
- 顶部状态同 Hub（路线标题、回合、耐力、心情、金币）
- 左侧面板：同 Hub 的角色属性面板
- **右侧列出五种训练**，每种显示：
  - 训练名称：`力量训练` `体力训练` `韧性训练` `集中训练` `保护训练`
  - 等级：如 `Lv.1`
  - 失败率：如 `失败率 0%`、`失败率 12%`
  - 选中后左侧显示蓝色预计属性增长：如 `+20` `+8` `+25`
- 底部中央：训练 EXP/熟练度，如 `133%`
- 支援卡头像（随机出现）和羁绊条
- 彩色光环（蓝色/金色/彩虹，稀有出现时提高优先级）

**训练属性映射**：
| 训练名 | 主要提升 | 次要提升 |
|--------|---------|---------|
| `力量训练` | 力量（大） | 保护（小） |
| `体力训练` | 体力（大） | 保护（小） |
| `韧性训练` | 保护（大） | 力量+体力（小） |
| `集中训练` | 集中/命中（大） | 力量（小） |
| `保护训练` | 保护/抗性（大） | 体力（小） |

**策略规则（已确认）**：
- 耐力管理：最大 100，每次训练消耗 15
- 耐力 ≤50 时失败率开始上升
- 失败率 ≥ 25% → 不训练，去休息
- 失败率 < 25% 时训练是默认选择
- 前 12 回合（第一营地前）：优先 `力量训练` 和 `体力训练`，`韧性训练` 为次
- 第一营地：`韧性`/`集中`/`保护` 有加成
- 第二营地：攻击型选海滩（攻击+集中加成），体力型选温泉（体力+保护加成）
- 支援卡羁绊目标：第一营地前三张主属性支援达绿色羁绊，营地结束时达橙色
- 两个以上彩色光环时可忽略失败率阈值

**❗需要从截图标定的区域**：
- 五种训练的卡片区域（名称、等级、失败率文字）
- 失败率数字的 OCR 区域
- 预计属性增长数字的 OCR 区域
- 支援卡头像区域和羁绊条颜色检测区域
- 彩色光环检测区域
- 训练 EXP 文字

## 4. 已确认的全局策略规则汇总

### 行动优先级（Training Hub 层）
```
红字委托可用？ → 做委托
   ↓ 否
失败率 ≥ 25%？ → 休息（60金币优先，不够用30金币）
   ↓ 否
训练（默认）
```

### 休息选择
```
金币 ≥ 60？ → 选项3（冥想室，60金币）
金币 < 60？→ 选项2（30金币）
（选项1 基本不用）
```

### 训练选择策略
- 耐力 ≤50 → 失败率开始上升
- 失败率 ≥25% → 不训练
- 前12回合：优先主属性训练 + 体力训练
- 第一营地：韧性/集中/保护有加成
- 第二营地：攻击型选海滩，体力型选温泉
- 两个以上彩色光环 → 可忽略失败率风险

### 耐力管理
- 最大 100，每次训练消耗 15
- （训练后耐力降低由状态追踪处理）
